📋 Lesson Canvas — 기획서 (Claude Code용)
프로젝트 개요
기존 프로젝트: 실시간 협업 화이트보드 (collab-whiteboard)
전환 방향: 선생님이 수업 중 실시간으로 사용하는 교육용 판서 + 문서 제작 도구
핵심 컨셉

선생님이 수업 자료를 실시간으로 만들면서 학생들이 동시에 화면을 보는 구조
단순 칠판 대체가 아닌 페이지 단위 문서 편집기 (인디자인/피그마 참고)
수업 후 PDF로 출력/저장 가능
대상 학년 제한 없음


역할 구조
기존 vs 변경
기존: 모든 참여자가 동등하게 그리는 협업 화이트보드

이미지 업로드 처리
피그마/노션 방식 채택

이미지 바이너리는 문서(Yjs)와 분리. 문서에는 URL과 hash만 저장.

흐름
선생님이 이미지 드래그앤드롭 or 파일 선택
    ↓
SHA-256 hash 계산 (클라이언트)
    ↓
POST /upload (multipart/form-data)
    ↓
서버: hash로 중복 체크 → 있으면 기존 URL 반환, 없으면 저장 후 URL 반환
    ↓
서버: /uploads/{hash}.{ext} 로컬 저장 (현재), 추후 S3/Supabase Storage 전환 가능
    ↓
클라이언트: ImageElement { src: URL, hash } 를 Yjs에 저장
    ↓
모든 학생: URL로 이미지 로드

PDF 출력
방식: html2canvas + jsPDF
ts// 출력 흐름
async function exportToPDF(pages: Page[]) {
  const pdf = new jsPDF({ format: "a4", orientation: "portrait" });

  for (let i = 0; i < pages.length; i++) {
    // 각 페이지의 fabric.js 캔버스를 이미지로 변환
    const dataUrl = fabricCanvas.toDataURL({ format: "png", multiplier: 2 });
    pdf.addImage(dataUrl, "PNG", 0, 0, 210, 297); // A4 mm
    if (i < pages.length - 1) pdf.addPage();
  }

  pdf.save(`${roomName}_수업자료.pdf`);
}

multiplier: 2 — 레티나 해상도로 렌더링

페이지 크기에 따라 jsPDF format 분기 ("a4" / [width, height])

주의사항 / 제약

이미지는 Yjs에 base64로 저장하지 않는다 — URL + hash만 저장, 바이너리는 서버에
페이지 이동은 선생님만 — 학생은 Yjs meta의 currentPage를 구독만 함
fabric.js와 Yjs 동기화 — fabric.js의 object:modified 이벤트 → Yjs 업데이트, Yjs 업데이트 → fabric.loadFromJSON() 방식으로 연동
as never 타입 우회 제거 — Phase 1에서 소켓 타입 정의 완성하면서 같이 정리
PDF 출력 시 — fabric.js의 toDataURL()을 직접 사용 (html2canvas로 DOM 캡처하는 것보다 품질 좋음)
